# Acceptance Check — AI 交付驗收 4 維度

W3 教學重點。AI 給結果後，主管必須會自己 sanity check，不能「AI 做完 = 結案」。

## 何時啟動

用戶說：
- 「這樣 OK 嗎？」
- 「怎麼驗收 AI 結果？」
- 「AI 給我的東西可以用嗎？」
- 「我要不要 own 這結果？」
- 「risk 評估」
- 「直接給客戶？」
- 「能不能交差？」

## 4 維度評分（每維 1-5 分）

### 1. 正確性 (Correctness)
> 「這個答案在事實 / 邏輯上對嗎？你怎麼知道？」
> 評分：5 = 100% 確定對 / 3 = 大部分對但有 1-2 點要再 verify / 1 = 看起來 plausible 但 zero verification

### 2. 完整性 (Completeness)
> 「該講的有都講到嗎？有沒有漏掉關鍵 caveat / 例外？」
> 評分：5 = 全面 / 3 = 主軸對但 edge case 不齊 / 1 = 缺重要面向

### 3. 風險暴露 (Risk Exposure)
> 「這結果被誰看到會出事？對外發出去後撤得回嗎？」
> 評分：5 = 純內部、撤得回 / 3 = 對外但小眾 / 1 = 對客戶 / 公開 / 不可逆

### 4. Own 結果 (Ownership)
> 「我願意把這直接掛我的名嗎？我願意 defend 這個 output 嗎？」
> 評分：5 = 完全願意 / 3 = 願意但要簽前再 review / 1 = 不願意

## 評分後決策

| 總分 | 動作 |
|------|------|
| 16-20 | 🟢 可直接用 / 出去 |
| 11-15 | 🟡 需 1 輪 refine，主管自己決定哪維補 |
| 6-10 | 🟠 risk 高，建議 reframe task 或部分接 / 部分自寫 |
| 4-5 | 🔴 不該交給 AI 這 task，回去 W3 Triage 表重看 |

## 反模式

- ❌ 不評分直接接受 AI output → W3 課程核心避免的
- ❌ 4 維度問完忘了給 verdict → 一定要結尾「總分 X，建議 Y」
- ❌ 主管不確定怎麼評 → 引導他用具體 stake 思考（「客戶看到會怎想」「老闆看到會怎想」）
