# W0 對話啟動卡 — 第一次跟 Claude 對話的 prompt 範本

> 用途：W0 共學裝機 session，你第一次跟 Claude 對話時用
> 怎麼用：依你的角色找對應那一套，挑 2 個 prompt 跑，或自由發揮

---

## ⚠️ 資料安全（先讀，30 秒）

啟動卡的 prompt 會碰到 1-on-1 / 客戶 email / 財務 / 客服 等場景，**不是所有資料都能貼進 Claude**

| ✅ 可直接貼 | 🟡 需匿名 / 去識別後貼 | 🔴 不可貼 |
|------------|--------------------|----------|
| 個人 prompt / 草稿 | 採訪逐字稿（去除受訪者真名）| 員工真實 1-on-1 紀錄（姓名 / 績效 / 薪資）|
| 公開內容 / 文章草稿 | 客戶 email（去除公司名 + 個資）| 360 feedback 真實內容（reviewer 匿名也敏感）|
| 範例資料 / Mock data | 數據趨勢（去除絕對金額）| 財務報表絕對數字（營收 / 成本）|
| 你個人的工作場景描述 | 採訪重點（去除未公開資訊）| 個資身分證 / 卡號 / 帳號 |

規則：
- 不確定 → 預設不貼，問講師
- prompt 是「框架」，真實 context 你自己決定要不要餵
- 也可以用 mock data / 改編版練習，不要因為「沒資料能貼」就跳過

> Anthropic Claude 商用 / 企業方案預設不用對話訓練模型，但企業合規層仍要嚴格 — 一個流出比技術問題嚴重 100x

---

## 找你的那一套（依角色自己對號）

| 套 | 適合角色 | 重點 |
|---|---------|------|
| **A 高層** | 總經理 / 副總 / 執行長層級 | 戰略 / 跨部門 / 董事會視角 |
| **B 內容創作** | 採訪 / 編輯 / 文案 / 行銷 | 摘要 / 標題 / 企劃 / repurpose |
| **C 業務電商** | 業務 / 電商 / 會員經營 | 客戶 read-out / funnel / 客服 / 自動化 |
| **D 人資財務設計** | 人資 / 財務 / 品牌設計 | 1-on-1 signal / 預算 / 品牌 audit / brief |

不確定自己屬哪套 → 挑最接近你日常工作的，跨套挑也完全可以

## 套 A：高層 — 戰略 / 跨部門 / 董事會視角

**A1. 戰略文件 stress test**
```
我手邊有一份 [部門名稱] 2026 Q3 策略 OKR 草稿（[手動貼上 3-5 行重點 OR 上傳檔案]）。
請扮演會問尖銳問題的 board member，挑出 3 個我沒想到的盲點。
不要安慰我，越尖銳越好。
```

**A2. 跨部門報告綜合**
```
我這週收到 4 份不同中心的進度報告（[手動貼上各報告開頭 3 行]）。
幫我整合出：
1. 跨部門共同 risk 是什麼
2. 我作為 [職位] 該主動 push 哪 2 件事
3. 哪些可以放著不管（identify what NOT to do）
```

**A3. 季度議題收斂**
```
2026 Q3 內部會議我要 framing 一個核心議題：[核心 topic 1 句話]。
請給我 5 個 framing 候選，每個從不同角度切入（用戶 / 競品 / 內部組織 / 長期戰略 / 短期執行）。
標出最容易讓決策者「聽得進去」那個 + 為什麼。
```

**A4. 模糊問題反問釐清**
```
我現在卡住的真實問題是：「[模糊一句話，例如：『我們的訂閱業務感覺後勁不足』]」
請不要直接給答案。反問我 5 個問題幫我把問題具體化，
我答完後你再給觀察。
```

**A5. 跨年度趨勢消化**
```
2025 年 AI 對親子教育產業最大的 3 個變化是什麼？
（不要 generic 答案，給我 5 個具體案例：哪間公司 / 做了什麼 / 對我們的啟示）
最後給我 1 個 Q4 該優先觀察的指標。
```

## 套 B：內容創作 — 採訪 / 編輯 / 文案 / 行銷

**B1. 採訪逐字稿 → 摘要 + 標題**
```
我有一份 [採訪對象] 的採訪逐字稿（[貼 200-500 字]）。
請幫我：
1. 5 行內 elevator summary
2. 3 個候選標題（一個正經 / 一個情緒 / 一個 SEO 友善）
3. 找出 3 個還沒被問深的點（後續 follow up 用）
```

**B2. 長稿冗詞偵測**
```
這篇 5000 字稿件（[貼前 500 字]）讀起來有點冗。
請找出 5 個你覺得最該砍掉 / 改寫的句子，並說明為什麼（拖節奏 / 重複 / 訊息密度低 / 等）。
不要重寫整篇，只要點出哪幾句。
```

**B3. 標題優化（A/B 候選）**
```
我這篇文章標題目前是「[現有標題]」。內文核心是 [1 句話]。
請給 5 個候選新標題，標明各自的「情緒鉤子 vs 訊息密度」trade-off。
不要選你自己最喜歡的 — 給我各種 framing。
```

**B4. 內容企劃 brainstorm**
```
2026 Q3 我想做一個 [主題] 內容專題，目標讀者是 [TA 描述]。
請給我 5 個切入角度，每個附：
- 1 句 hook
- 適合的內容格式（長文 / 短影音 / podcast / interactive）
- 預期讀者反應
```

**B5. 跨平台 repurpose**
```
我有一篇 3000 字的長文（[貼前 200 字 + 結論]）。
幫我改寫成：
1. LINE 群轉發 300 字版（家長社群）
2. IG 貼文 5 張卡片文案
3. Newsletter 開頭 1 段
保留原作者語氣，不要 buzz word。
```

## 套 C：業務 / 電商 / 會員經營

**C1. 客戶 email → 戰略 read-out**
```
我收到一份大客戶的 email（[貼內容 3-5 段]）。
請幫我：
1. 客戶真正想要什麼（表面 vs 真實需求）
2. 哪些訊號暗示 next step（升級 / 流失 / 觀望）
3. 我該回什麼讓對話往前推（給 3 個語氣選項）
```

**C2. 銷售 funnel 數據解讀**
```
我這季的 funnel 數字是：[手動貼數字，例如 訪問 10K → 註冊 1K → 付費 100 → 續訂 60]
請：
1. 找出最大的漏水段
2. 提 3 個快速可測試的假設（為什麼漏 + 怎麼測）
3. 哪個假設最值得這週試（cost / impact ratio）
```

**C3. 顧客體驗 audit**
```
我手邊有 20 則客服紀錄（[貼 5-10 則 keyword]）。
幫我：
1. 分類 5 大 pain point + 出現頻率
2. 哪些是「真正影響續訂」哪些是 noise
3. 哪 2 個如果今天就修，下季 NPS 會動
```

**C4. 數據儀表板 spec**
```
我想建一個追蹤 [業務目標] 的儀表板。
請問我 5 個問題幫我把 spec 釐清，
我答完你再給我建議：要哪幾個指標 / 拉式還是推式 / 多久看一次。
```

**C5. 業務流程自動化提案**
```
我團隊目前最浪費時間的 3 件事是：
1. [手動 type]
2. [手動 type]
3. [手動 type]
針對每件事，給我：
- AI 能完整接掉嗎 / 50% 還是不行
- 如果可以接，第一步從哪個 90 分鐘 prototype 開始
- 風險（AI 做錯什麼成本最高）
```

## 套 D：人資 / 財務 / 設計

> 多元職能合一套，選最相關的 2 個跑

**D1. 1-on-1 紀錄 → morale signal（人資）**
```
我這個月做了 6 場員工 1-on-1 的紀錄（[貼 6 段 3-5 行 summary，去除真實姓名]）。
請找出：
1. 跨員工的共同 pain point（不是個別案例）
2. 哪些訊號可能預示流失風險
3. 我作為 HR 該主動 surface 給 [部門主管 / 高層] 的 1 件事
```

**D2. 預算分析 / 財務數據（財務）**
```
我有這份 2026 Q1-Q2 部門費用 vs 預算 對照（[貼數字，可改成相對比例]）。
請：
1. 找出 3 個最該關注的差異（不是最大的，是「該說明的」）
2. 哪些是 one-off 哪些是 trend
3. 我準備季度報告，該 framing 成「執行落實」還是「需要調整 budget」？
```

**D3. 設計品牌一致性 audit（品牌設計）**
```
我們品牌最近在這 3 個地方出現（[貼 URL / 圖片描述]）。
請幫我審：
1. 品牌調性是否一致（文字 / 視覺 / 互動）
2. 哪一個最「不像我們」
3. 修哪一個 ROI 最高（最多人看到 / 最影響感受）
```

**D4. 設計 brief 反問釐清（品牌設計）**
```
業務同事丟給我一個 design brief：「[貼 1-2 句模糊 brief]」
請扮演接案 designer，反問我 5 個問題（受眾 / 目的 / scope / 風格參考 / 成功標準），
不要直接設計。
```

## 怎麼跑（session 內約 14 分鐘）

```
0:51-0:53  從你的套（A/B/C/D）挑 1 個 prompt → 貼進 Claude
0:53-0:58  讀 Claude 回應，追問 1 輪深入
0:58-1:00  挑第 2 個 prompt（同套或跨套）
1:00-1:05  追問 1 輪 + 截圖「比預期好」或「比預期差」的瞬間
```

- 比預期**好**的 → 截圖發 LINE 群 #wow
- 比預期**差**的 → 截圖發 LINE 群 #disappointment（W3 我們會用這些來練 AI 驗收）
